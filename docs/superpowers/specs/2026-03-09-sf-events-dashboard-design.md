# SF Events Dashboard — Design Spec

## Overview

A locally-run Next.js app that continually discovers events in San Francisco / Bay Area, scores them against a learned preference profile, and displays them on a vertical TV screen. A separate dashboard allows the user to rate events (thumbs up/down), manage sources, edit preferences, and add events to Google Calendar.

**User location:** 1550 Mission St, San Francisco (has a car)

## Architecture

Single Next.js 14+ app (App Router, TypeScript) running locally on Mac.

### Three layers:

**1. Data Collection** (runs every few hours via node-cron)

- **Platform discovery:** Claude API calls with web search tool enabled. For each platform source, send a query like "Find upcoming events in San Francisco this week on Eventbrite" — Claude searches the web and returns structured event JSON directly.
- **Venue tracking:** Fetch the HTML of known venue event pages (e.g., Manny's, KQED), send the HTML to Claude for structured extraction.
- Both paths output the same event structure (see Data Model below).
- Events are deduplicated by title + date + venue (fuzzy matched by Claude during extraction).

**2. Intelligence**

- Each new event is scored 0–100 by Claude against the user's preference profile.
- **Preference profile:** A natural-language text document summarizing what the user likes and dislikes. Stored in SQLite, viewable and editable on the dashboard.
- Profile is regenerated by Claude after interactions accumulate, analyzing the full interaction history.
- **Composite score:** relevance (0–100 from Claude) minus distance penalty (~1 point per mile, max -15) plus recency boost for sooner events.
- Distance computed via Nominatim geocoding (free, no API key).

**3. Display**

- `/tv` — Read-only vertical full-screen display for the TV
- `/dashboard` — Interactive control panel for phone/laptop

**Seed preference profile** (based on past events attended):

> Enjoys live podcast tapings and media events (e.g., KQED Close All Tabs). Interested in comedy and improv, especially with a startup/tech angle (e.g., Manny's One Hour Startup). Open to a wide range of events — cast a wide net initially.

## TV Display (`/tv`)

**Layout: Day-Grouped Timeline** (dark theme, vertical orientation)

- Events grouped by day: "Today — Mon Mar 9", "Tomorrow — Tue Mar 10", etc.
- Each event row shows: time, title, venue, distance, price, description snippet, and color-coded relevance score badge (green 80+, blue 60–79, yellow 40–59, red <40).
- Left border color matches the score badge color.
- 14 days of detailed events displayed.
- "Coming Up" section at the bottom for events beyond 14 days — compact rows showing just date, title, and score.
- Auto-refreshes periodically to pick up new data.
- Auto-scrolls or pages through the list so the full 14 days are visible over time.

## Dashboard (`/dashboard`)

**Layout: Day-Grouped Card Feed** (light theme)

- Same day grouping as the TV display for mental model consistency.
- Each event is a card showing: title, date/time, venue, distance, price, description, and score badge.
- **Action buttons per card:**
  - Thumbs up (interested) — records positive interaction
  - Thumbs down (not for me) — records negative interaction, card fades out
  - Calendar button — opens Google Calendar URL with pre-filled event details (title, date/time, location, description + link). No API key needed.
- Filter pills: All, Today, This Week, Unrated
- **Navigation tabs:**
  - **Events** — the main card feed
  - **Sources** — manage event sources (add/remove/enable/disable). Each source has a name, type (platform or venue), and either a search query or direct URL.
  - **Profile** — view and edit the preference profile text. Shows interaction history.

## Data Model (SQLite + Drizzle ORM)

### events
| Column | Type | Description |
|--------|------|-------------|
| id | integer PK | Auto-increment |
| title | text | Event title |
| date | text | Event date (YYYY-MM-DD) |
| time | text | Event time (HH:MM) |
| venue | text | Venue name |
| address | text | Full address |
| lat | real | Latitude (from Nominatim) |
| lng | real | Longitude (from Nominatim) |
| distance_miles | real | Distance from 1550 Mission St |
| description | text | Event description |
| price | text | Price info (e.g., "Free", "$15", "$20–$45") |
| url | text | Link to original event page |
| image_url | text | Event image URL (nullable) |
| source | text | Source name (e.g., "Eventbrite", "Manny's") |
| relevance_score | integer | 0–100 from Claude |
| final_score | integer | Composite score (relevance - distance + recency) |
| created_at | text | ISO timestamp |
| expires_at | text | ISO timestamp (event date + 1 day) |

### sources
| Column | Type | Description |
|--------|------|-------------|
| id | integer PK | Auto-increment |
| name | text | Display name |
| type | text | "platform" or "venue" |
| url | text | Direct URL for venue sources (nullable) |
| search_query | text | Search query for platform sources (nullable) |
| enabled | integer | 1 = active, 0 = disabled |
| last_scraped_at | text | ISO timestamp (nullable) |

### interactions
| Column | Type | Description |
|--------|------|-------------|
| id | integer PK | Auto-increment |
| event_id | integer FK | References events.id |
| action | text | "thumbs_up", "thumbs_down", or "calendar_added" |
| created_at | text | ISO timestamp |

### preference_profile
| Column | Type | Description |
|--------|------|-------------|
| id | integer PK | Auto-increment |
| profile_text | text | Natural language preference description |
| version | integer | Incrementing version number |
| created_at | text | ISO timestamp |

## Tech Stack

- **Next.js 14+** — App Router, TypeScript
- **SQLite** — via Drizzle ORM
- **Anthropic Claude API** — web search tool for platform discovery, extraction from venue HTML, event scoring, preference profile generation
- **Nominatim** — free geocoding for distance calculation
- **Google Calendar URL scheme** — pre-filled calendar links, no API key needed
- **node-cron** — scheduled scraping every few hours
- **Tailwind CSS** — styling

## Environment

- Runs locally on Mac
- Single env var: `ANTHROPIC_API_KEY`
- TV connects to `http://localhost:3000/tv`
- Dashboard accessed at `http://localhost:3000/dashboard`
- Optional: LaunchAgent for auto-start on boot
